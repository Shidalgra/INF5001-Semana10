Es totalmente normal que se sienta extraño pasar de un mundo tan estructurado como el de SQL 
(donde todo son tablas 📊, filas, columnas y claves foráneas) a uno NoSQL como MongoDB.

Para que tu mente haga el puente rápido entre lo que ya conoces y lo nuevo, mira esta equivalencia básica:

* Base de datos 🗄️ ➔ Sigue llamándose Base de datos.

* Tabla 📋 ➔ En MongoDB se llama Colección (Collection).

* Fila / Registro 📑 ➔ En MongoDB se llama Documento (Document) y se guarda en un formato muy parecido a JSON.

La gran diferencia es que en MongoDB no hay "tablas relacionales" estrictas ni esquemas fijos obligatorios 
por defecto; cada documento puede tener campos diferentes si así se requiere.

Una app de lista de tareas (To-Do List) avanzada nos permitirá modelar datos fácilmente y entender cómo guardar,
leer, actualizar y borrar documentos en MongoDB usando Node.js.

De acuerdo con el material de la Semana 10, todo proyecto profesional de Node.js debe comenzar preparando su entorno
y su "cédula de identidad".

Para poner las manos en el código, el primer paso es crear una carpeta vacía en tu computadora para el proyecto 
(por ejemplo, todo-mongo-app), abrir la terminal dentro de esa carpeta y prepararla para usar paquetes de NPM.
